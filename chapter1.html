Chapter-1:Database management system

Data

Data refers to raw facts and figures without any context. It can be numbers, characters, or symbols collected for analysis and processing.

Information

Information is processed data that is meaningful and useful for decision-making.

Features of Information

Information should be accurate, relevant, timely, complete, and understandable.

Database and its Purpose

A database is an organized collection of data that allows easy access, management, and updating. Its purpose is to store and retrieve data efficiently.

Terminologies in Database

  • Table: Collection of related data entries organized in rows and columns.
  • Field: A single piece of data or attribute in a table (column).
  • Record: A row in a table representing a single data item.
  • Tuple: Another term for a record or row in relational databases.
  • Object: In object-oriented databases, it is an instance of a class containing data and behavior.
  • Keys: Attributes used to identify records uniquely (e.g., primary key).

Data Dictionary

A data dictionary contains metadata — data about data — describing the structure, relationships, and constraints of data in a database.

Database Management System (DBMS)

Introduction

A DBMS is software that interacts with users, applications, and the database itself to capture and analyze data.

Objectives

  • To provide a convenient and efficient environment for users to store and retrieve data.
  • To ensure data integrity and security.
  • To manage data efficiently and provide multi-user access.

Advantages

  • Data sharing and security.
  • Reduced data redundancy.
  • Improved data integrity and consistency.

Disadvantages

  • Cost of hardware and software.
  • Complexity in design and maintenance.
  • Potential for system failures affecting all users.

Types of Database Model

Hierarchical

Data is organized in a tree-like structure with parent-child relationships.

Network

Data is organized using a graph structure allowing many-to-many relationships.

Relational

Data is organized into tables (relations) with rows and columns; uses SQL.

Entity Relational

Uses diagrams to model data entities and relationships visually.

Integrity Constraints and Types

Domain

Specifies the permissible values for a given attribute.

Entity

Ensures each entity has a unique identifier.

Referential

Maintains consistency among tuples in different relations using foreign keys.

Keys

Uniquely identify tuples in a table, e.g., primary key, candidate key.

Normalization

Introduction

Process of organizing data to minimize redundancy and improve data integrity.

Normal Forms

Normal Forms
Form Description
1NF (First Normal Form) Eliminates duplicate columns; ensures atomicity of data.
2NF (Second Normal Form) Removes subsets of data that apply to multiple rows; ensures full functional dependency.
3NF (Third Normal Form) Removes columns that are not dependent on the primary key.

Advantages

  • Reduces data redundancy.
  • Improves data integrity.
  • Facilitates efficient updates.

Disadvantages

  • Complex queries due to many tables.
  • Performance overhead from joins.

Centralized and Distributed Database

Introduction

Centralized database stores data at a single location; distributed database stores data across multiple locations.

Advantages

  • Centralized: Easier management and control.
  • Distributed: Improved reliability and faster access locally.

Disadvantages

  • Centralized: Single point of failure and bottleneck.
  • Distributed: Complexity in synchronization and management.

Comparison

Centralized vs Distributed Database
Feature Centralized Distributed

Access Speed Slower for remote users Faster locally

Database Security

Introduction

Database security involves protecting data from unauthorized access, misuse, or threats.

Challenges

  • Unauthorized access and hacking.
  • Data breaches and leaks.
  • Backup failures and data loss.

Security Measures

  • Authentication and access control.
  • Encryption of sensitive data.
  • Regular backups and audits.

Roles of DBA

  • Manage database security and integrity.
  • Perform backups and recovery.
  • Optimize performance and manage access.

Practical Topics: DDL and DML Languages

DDL (Data Definition Language): Commands that define database structure (e.g., CREATE, ALTER, DROP).

DML (Data Manipulation Language): Commands to manipulate data (e.g., INSERT, UPDATE, DELETE, SELECT).

Q4. What is a relational database? How is it different from other database models?

Ans: A relational database stores data in tables, where each row is a record and each column is an attribute. It uses SQL for data manipulation. It differs from other models by its use of table format, strong data integrity, and ease of querying using relational algebra.

Ans: Data redundancy means having duplicate data in multiple places. DBMS reduces redundancy by storing data in a centralized system, using normalization techniques, and avoiding unnecessary repetition through relational design.

Q7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

Ans: A Database Administrator (DBA) is a person responsible for managing and maintaining the database system.

Responsibilities include:

  • Database installation and configuration
  • Ensuring data security and backup
  • Monitoring database performance
  • User management and access control

Q8. Define normalization. Explain 1NF, 2NF and 3NF with suitable examples. Explain the normalization process with examples.

Ans: Normalization is the process of organizing data to reduce redundancy and improve data integrity.

1NF (First Normal Form): Ensures each column has atomic values (no repeating groups).

2NF (Second Normal Form): 1NF + no partial dependency (non-key attribute depends on full primary key).

3NF (Third Normal Form): 2NF + no transitive dependency (non-key attribute should not depend on other non-key attributes).

Q9. Explain the terms: primary key, foreign key, and candidate key with examples.

Ans:

  • Primary Key: A unique identifier for each record in a table. Example: StudentID in Student table.
  • Foreign Key: A field in one table that refers to the primary key in another table. Example: StudentID in Marks table.
  • Candidate Key: All possible fields that can uniquely identify records. Example: Email and StudentID can both be candidate keys.

Q10. What is SQL? Explain its components and common functions.

Ans: SQL (Structured Query Language) is a language used to manage and manipulate relational databases.

Components of SQL:

  • DDL (Data Definition Language): CREATE, ALTER, DROP
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE
  • DCL (Data Control Language): GRANT, REVOKE
  • TCL (Transaction Control Language): COMMIT, ROLLBACK

Common SQL Functions: COUNT(), AVG(), MAX(), MIN(), SUM(), NOW()

Defination

a. Data Dictionary: A data dictionary is a centralized repository that stores metadata—information about data—such as data types, relationships, constraints, and descriptions of tables, columns, and other database objects.

b. Primary Key: A primary key is a field (or combination of fields) in a table that uniquely identifies each record. It cannot contain null values and must have unique values for each row.

c. Relationship: A relationship in a database refers to how data in one table is related to data in another. Common types include one-to-one, one-to-many, and many-to-many relationships.

d. Data Manipulation Language (DML): DML is a subset of SQL used to retrieve, insert, update, and delete data in a database. Common DML commands are SELECT, INSERT, UPDATE, and DELETE.

e. Structured Query Language (SQL): SQL is a standard language used to interact with relational databases. It allows users to create, manage, and manipulate data and database structures.

f. Data Integrity: Data integrity refers to the accuracy, consistency, and reliability of data stored in a database. It is enforced through constraints like primary keys, foreign keys, and validation rules.

g. Data Definition Language (DDL): DDL is a subset of SQL used to define and manage database structures such as tables, indexes, and schemas. Common DDL commands include CREATE, ALTER, and DROP.

h. Data Security: Data security involves protecting data from unauthorized access or corruption. This includes user authentication, encryption, and access control policies.

i. Database System: A database system is a combination of the database itself, the DBMS software, and the applications that interact with it. It is designed to store, manage, and retrieve data efficiently and securely.

Data Constraints

Data constraints are rules applied to table columns to enforce data integrity:

  • NOT NULL: Ensures that a column cannot have a NULL value.
  • UNIQUE: Ensures all values in a column are different.
  • PRIMARY KEY: Uniquely identifies each record in a table and cannot be NULL or duplicate.
  • AUTO INCREMENT: Automatically increases the value of a column (usually used for IDs).

Operators (AND, OR, NOT)

These are logical operators used to filter records in SQL queries:

  • AND: Returns true if both conditions are true.
  • OR: Returns true if at least one condition is true.
  • NOT: Reverses the result of a condition (returns true if condition is false).

WHERE Clause

The WHERE clause is used to filter records that meet a specific condition.

Example: SELECT * FROM users WHERE age > 18;

ORDER BY

The ORDER BY clause is used to sort the result set in ascending (default) or descending order.

Example: SELECT * FROM users ORDER BY name ASC;

SQL JOINs

SQL JOINs are used to combine rows from two or more tables based on a related column between them.

  • INNER JOIN: Returns only the rows that have matching values in both tables.
    Example:
    SELECT employees.name, departments.name
    FROM employees
    INNER JOIN departments ON employees.dept_id = departments.id;
  • LEFT JOIN (or LEFT OUTER JOIN): Returns all rows from the left table, and the matched rows from the right table. If no match, NULL is returned from the right side.
    Example:
    SELECT employees.name, departments.name
    FROM employees
    LEFT JOIN departments ON employees.dept_id = departments.id;
  • RIGHT JOIN (or RIGHT OUTER JOIN): Returns all rows from the right table, and the matched rows from the left table. If no match, NULL is returned from the left side.
    Example:
    SELECT employees.name, departments.name
    FROM employees
    RIGHT JOIN departments ON employees.dept_id = departments.id;
  • FULL JOIN (or FULL OUTER JOIN): Returns all rows when there is a match in either left or right table. Rows without matches in one of the tables will have NULLs.
    Example:
    SELECT employees.name, departments.name
    FROM employees
    FULL OUTER JOIN departments ON employees.dept_id = departments.id;
